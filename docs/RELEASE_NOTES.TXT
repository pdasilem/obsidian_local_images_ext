0.19.0
Refactor: Unified attachment processing around canonical naming and target-state layers for both local attachments and external media links.
Fix: Local attachment links are rewritten only after successful file apply, preventing file/link desynchronization during rename and move operations.
Fix: External media processing now uses the same target-state/apply pattern as local attachments.
Fix: Oversized symlink/junction routing now uses a normalized target-directory check instead of a fragile path substring guard.

0.18.1
Fix: Automatic processing for existing notes now runs a follow-up reconcile pass so newly inserted attachments are also moved into their final target folder, including oversized destinations.
Docs: Added README notes about oversized attachment routing, note-name attachment renaming, and BRAT installation.

0.18.0
Add: Optional symlink/junction mode for the oversized media subfolder. When enabled, oversized files are moved into that folder via direct filesystem operations instead of the Obsidian vault adapter.
Fix: Oversized subfolder defaults and UI now use `big` without a leading slash.
Refactor: Removed legacy and dead code around attachment naming and file-move state.

0.17.10
Fix: Local attachment processing now applies the computed target path directly, so files are moved to the correct folder, including the oversized media subfolder, instead of only being renamed.

0.17.9
Fix: Oversized media routing now compares file sizes in bytes, making large local attachments move to the oversized subfolder reliably.
Change: Renamed the auto-processing setting for existing notes to clarify that it applies to new attachments and media links added to existing markdown notes.

0.17.8
Fix: Existing notes now process newly added media links and attachments from markdown modifications instead of relying on the 1000 ms file-creation window.
Fix: Local oversized attachment detection now uses a size probe based on the configured upper size limit, so manual processing places large files into the oversized subfolder correctly.

0.17.7
Fix: Reprocessing no longer renames attachments again when Note name + counter naming is enabled and the current file name already matches the note-name pattern.

0.17.6
